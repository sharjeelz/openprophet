OpenProphet

Autonomous AI options trading agent.
Powered by Claude + Alpaca. Runs 24/7. No human needed.

Risk Warning

This is not financial advice. Options trading involves substantial risk of loss. Paper trading is recommended. You are solely responsible for any trades executed and financial losses incurred.

How It Works

1
Heartbeat Loop
Claude wakes up automatically on a schedule tied to market hours. Each wake-up is called a beat. Between beats the agent is idle — no CPU, no cost.
2
Research & Analysis
Each beat Claude fetches live news, MarketWatch headlines, technical indicators (RSI, MACD), options chains, and account status via 45+ MCP tools.
3
Strategy Rules
Claude follows TRADING_RULES.md — instruments, DTE range, delta, position sizing (5% per trade), stop loss (-25%), profit target (+40%), max 3 trades/session.
4
Order Execution
When a setup passes all filters, Claude places a limit options order on Alpaca at mid-price. Contract quantity is calculated from your real account balance automatically.
5
Position Management
Every beat Claude checks open positions. Stop hit (-25%), target hit (+40%), or thesis broken → it closes the position. No manual intervention needed.

Heartbeat Schedule

The agent wakes up more frequently during high-activity market hours and slows down when markets are quiet. All times are US Eastern.

Pre-Market 4:00–9:30 AM Every 15 min — gather intelligence, review overnight news, plan the session
Market Open 🔥 9:30–10:30 AM Every 2 min — highest frequency. Best window for momentum entries. Agent most active here.
Midday 10:30 AM–3 PM Every 10 min — monitor positions, watch for VWAP setups, manage stops
Market Close 🔥 3:00–4:00 PM Every 2 min — decide holds vs. closes, manage end-of-day risk
After Hours 4:00–8:00 PM Every 30 min — review session, log activity, assess overnight holds
Markets Closed Overnight Every 1 hour — minimal activity, monitors for major news events
How to adjust: Go to Settings → Heartbeat to change the interval for any phase. Lower = more beats = more API cost. The agent can also adjust its own heartbeat dynamically using the set_heartbeat tool — for example speeding up during volatile periods.

User Manual

Starting & Stopping
▸ Press Start in the top bar to begin the heartbeat loop. The agent will fire its first beat within seconds.
▸ Press Stop to halt. Any in-progress beat is killed immediately. Open positions remain open on Alpaca.
▸ Press Pause to suspend the heartbeat without killing the current beat. Resume resumes from where it left off.
▸ The agent remembers its session across beats — context from earlier beats is carried forward automatically.
Sending Messages to the Agent
▸ Type in the Terminal chat bar at the bottom and press Enter to send a message mid-beat.
▸ If the agent is mid-beat, it will be interrupted, your message delivered, and it resumes on the same session.
▸ If idle, your message triggers an immediate ad-hoc beat before the next scheduled one.
▸ Example: "Don't trade today, markets are too volatile" or "Check if our NVDA position is still valid"
Using the Manager
▸ The Manager tab is a separate AI that does NOT trade. Use it for research, analysis, and configuration.
▸ Ask it to research a ticker: "Research NVDA — what's the setup today?"
▸ Ask it to update strategy: "Add crude oil ETFs to the instrument universe"
▸ Ask it to explain: "Why did the agent skip the XOM trade?"
▸ The Manager has access to all the same market data tools as the trading agent.
Changing Strategy Rules
▸ Go to Strategies tab and edit TRADING_RULES.md directly — or ask the Manager to update it for you.
▸ Changes take effect on the next beat. No restart needed.
▸ The rules control: what to trade, when to enter, how to size, when to exit.
▸ Keep rules specific and actionable — vague rules lead to vague behavior.
Permissions & Guardrails
▸ Settings → Permissions — hard guardrails enforced at the MCP level, before Claude even decides.
▸ Allow Live Trading — master switch. Off = agent can analyze but never place orders.
▸ Allow Options / Stocks — asset class switches. Disable stocks to force options-only.
▸ Max Order Value — cap the dollar value of any single order. Set to 0 for unlimited.
▸ Max Daily Loss % — if portfolio drops this % intraday, agent auto-pauses. Circuit breaker.
▸ Require Confirmation — blocks all orders with an error message, so you can review before enabling.
Viewing Results & Logs
▸ Terminal tab — full real-time log of every beat, tool call, and decision.
▸ Trades tab — every order the agent placed, with symbol, side, and price.
▸ Portfolio tab — live Alpaca account: balance, open positions, buying power.
▸ activity_logs/ folder — daily markdown journals the agent writes itself after each session.
▸ decisive_actions/ folder — detailed logs of every trade entry and exit with full reasoning.
Tips
▸ Start with paper trading. Watch 2-3 sessions before enabling live funds.
▸ The agent is most useful during Market Open (9:30–10:30 AM ET) — that's when it finds the most setups.
▸ Use the Manager every morning to get a session watchlist before starting the agent.
▸ If the agent is overtrading, reduce Max Tool Rounds Per Beat in Settings or tighten the entry filters in TRADING_RULES.md.
▸ If the agent is undertrading (too passive), ask it via Terminal chat: "Be more aggressive today, look for setups actively."
▸ Check decisive_actions/ weekly to review what the agent did and why — this is your feedback loop.
